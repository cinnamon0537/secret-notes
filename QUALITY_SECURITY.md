# SonarQube and Snyk

## SonarQube

- Intended for static code analysis, code smells, duplication detection, and maintainability metrics.
- Expected to run in CI before build or together with lint.

## Snyk

- Intended for dependency and vulnerability scanning.
- Expected to fail the pipeline on critical unresolved findings.

## Current Status

- Neither tool is integrated yet.
- No live scan screenshots are available yet.

## Next Step

- Add SonarQube and Snyk to both CI pipelines once the servers or cloud integrations are available.
